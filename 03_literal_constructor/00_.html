생성자 사용을 자제하고 대신 리터럴 표기법을 사용하자
객체, 배열, 정규식 등의 리터럴을 다루며, 이에 대응하는 Object()나 Array()등의 내장 생성자 함수에 비해 리터럴 표기법을 쓰는 게 더 좋은 이유를 살펴본다.
사용자 정의 생성자에 대해서도 알아보고, 생성자가 의도대로 작동할 수 있도록 보장하는 방법을 살펴본다.

3.1 객체 리터럴

  • 자바스크립트에서 '객체'란 이름-값 쌍의 해시 테이블을 생각하면 된다.
  • 원시 데이터 타입, 객체, 함수 모두 값이 될 수 있고 값이 되는 함수를 메서드라고 한다.
// 빈 객체에서 시작
var dog = {};
 
// 프로퍼티 하나를 추가
dog.name = "Benji";
 
// 메서드 추가
dog.getName = function() {
    return dog.name;
};
cs
위 예제는 빈 객체에서 시작했고 프로퍼티와 메서드를 추가 프로그램 생명주기 중 어느 때라도 다음과 같은 일을 할 수 있다.
/ 프로퍼티와 메서드 값을 변경
dog.getName = function() {
    // 메서드가 하드코딩("Fido")된 값을 반환하도록 재정의
    return "Fido";
};
 
// 프로퍼티나 메서드 삭제
delete dog.name;
 
// 다른 프로퍼티나 메서드를 추가한다.
dog.say = function() {
    return "Woof!";
};
dog.fleas = true;
cs
반드시 빈 객체에서 시작해야 하는 것이 아니다. 객체 리터럴 표기법을 쓰면 다음 예제처럼 생성 시점에 객채에 기능을 추가 할 수 있다.
var dog = {
    name: "Benji",
    getName: function () {
        return this.name;
    }
};
cs
책에서 빈 객체표현은 간결성을 위한 것이며 자바스크립트에서는 빈 객체가 없다. {} 객체조차도 이미 Object.prototype에서 상속받은 프로퍼티와 메서드를 가진다. 비어있다는 말은 어떤 객체가 상속받은 것 외에는 자기 자신의 프로퍼티를 갖고 있지 않다는 뜻

객체 리터럴 문법

  • 객체를 중괄호({와})로 감싼다.
  • 객체 내의 프로퍼티와 메서드를 쉼표(,)로 분리한다. 마지막 이름-값 쌍 뒤에 쉼표가 들어가면 ***IE7이하***에서는 에러가 발생하므로, 마지막에는 사용하지 말아야 한다.
  • 프로퍼티명과 프로퍼티 값을 콜론(:)으로 분리
  • 객체를 변수에 할당할 때는 닫는 중괄호 뒤에 세미콜론(;)을 빼먹지 말자.

생성자 함수로 객체 생성하기

객체를 생성할 때는 직접 만든 생성자 함수를 사용할 수도 있고, Object(), Date(), String()등 내장 생성자를 사용할 수도 있다. 다음 예제는 동일한 객체를 생성하는 두 가지 방법

// 첫번째 방법 : 리터럴 사용
var car = {goes: "far"};
 
// 다른 방법 - 내장 생성자 사용 ( 안티 패턴 )
var car = new Object();
car.goes = "far";
cs
  • 객체 리터럴 표기법이 더 짧다.
  • 객체란 변경가능한 해시에 불과하며 특별한 것이 아니다.
  • 리터럴 표기법을 사용하면 유효범위 판별 작업도 발생하지 않는다.(위 예제에서 스코프내 Object()가 중복되는지 확인 한다는건가요?)

객체 생성자의 함정

문제의 기능은 Object() 생성자가 인자를 받을 수 있다는 점이다. 인자로 전달되는 값에 따라 생성자 함수가 다른 내장 생성자에 객체 생성을 위임할 수 있고, 따라서 기대한 것돠는 다른 생성자로 생성된 객체가 반환된다.
// 모두 안티 패턴
 
// 빈 객체
var o = new Object();
console.log(o.constructor === Object); // true
 
// 숫자 객체
var o = new Object(1);
console.log(o.constructor === Number); // true
console.log(o.toFixed(2)); // "1.00"
 
// 문자열 객체
var o = new Object("I am a string");
console.log(o.constructor === String); // true
console.log( typeof o.substring); // "function"
 
// 불린 객체
var o = new Object(true);
console.log(o.constructor === Boolean); // true
cs
o.constructor의 값이 다 다르다. new Object()를 사용하지 마라.

3.2 사용자 정의 생성자 함수

직접 생성자 함수를 만들어 객체 생성하기
var adam = new Person("Adam");
adam.say(); // "I am Adam"
cs

생성자 함수 정의

var Person = function(name) {
    this.name = name;
    this.say = function() {
        return "I am " + this.name;
    };
};
cs

new와 함께 생성자 함수를 호출하면 함수안에서 다음과 같은 일이 일어난다.

  • 빈 객체 생성. 이 객체는 this라는 변수로 참조할 수 있고, 해당 함수의 프로토타입을 상속받는다.
  • this로 참조되는 객체에 프로퍼티와 메서드가 추가된다.
  • 마지막에 다른 객체가 명시적으로 반환되지 않을 경우, this로 참조된 이 객체가 반환된다.

연쇄할당문에서의 변수

function foo(){
var a = b = 0;
}
cs

a는 지역변수가 되지만 b눈 전역 변수가 된다.

function foo(){
var a, b;
a = b = 0;
}
cs

변수를 미리 선언해두면 의도치 않은 전역 변수가 생성되는 일이 업서서, 연쇄 할당문을 사용해도 문제가 되지 않는다.

var 선언을 빼먹었을때의 부작용

  • 명시적으로 선언된 변수 : var 를 사용하여 명시적으로 선언된 전역변수는 삭제할 수 없다.
  • 암묵적 전역 변수 : var를 사용하지 않고 생성한 변수(함수안이든 아니든)는 삭제할 수 없다.

ES5 스트릭트 모드에서는 선언되지 않은 변수에 값을 할당하면 에러가 발생한다.

전역 객체에 대한 접근

브라우저에서는 코드 어느곳에서든 window속성을 통해 전역객체에 접근가능하다.
함수를 new와 생성자를 사용해 호출하지 않고 그냥 함수로 호출한 경우, 함수 안에서 this는 항상 전역객체를 가리키기 때문. (ES5 스트릭트 모드에서는 이 방법 불가능 - 이모드에서 실행될때에는 다른 패턴 사용)
예를 들어 라이브러리를 개발하고 있다면, 라이브러리 코드를 즉시 실행함수로 감싼 후, 즉시 실행함수의 인자로 전역 유효범위를 가리키는 this를 전달하는 방법이 있다.

단일 var 패턴

변수를 선언할 때 초기값을 주어 초기화하는 것 역시 좋은습관이다.
코드의 가독성도 향상되고, 나중에 코드를 볼 때 변수의 초기값에 근거해 변수의 용도가 무엇인지 짐작할 수 있다.

호이스팅(hoisting) : 분산된 var 선언의 문제점

hoisting이란 ? 함수 내 여리저기서 여러개의 var선언을 사용했지만 실제로는 모든 함수 상단에서 변수가 선언된 것과 동일하게 동작. 
// 안티패턴
myname = 'global'; //전역 변수
function func(){
alert(myname);	//undefined
var myname = "local"
alert(myname);	//local
}
cs
첫번째 alert 에서 undefined 이 나오는 이유 : myname 이 이 함수의 지역변수로 선언되었다고 간주하기 때문. 모든 변수 선언문은 상단으로 끌어올려진다. 이러한 혼란을 피하기 위해서는 변수를 모두 맨 첫 줄에서 선언하는 것이 좋다.
위에코드를 아래와 같이 동일하게 동작

myname = 'global'; //전역 변수
function func(){
var myname; // var myname = undefinde와 동일

2.4 for-in 루프

  • for-in 루프는 객체를 순회할 때 사용
  • 배열을 순회할때에도 쓸 수 있지만, 권장사항은 아니다.
  • 객체의 프로퍼티를 순회할 때는 프로토토타입 체인으르 따라 상속되느느 프로퍼티들을 걸러내기 위해 hasOwnProperty()메서드를 사용해야 한다.
  • 개발자가 코드에 확신을 가지고 있다면, hasOwnProperty()를 사용하지 않아도 되며, 이 경우 속도가 약간 개선됨

hasOwnProperty() 를 적용한 다양한 경우들

  • for(var i in man){
        if(man.hasOwnProperty(i)){
            console.log(i,":", man[i]);
        }
    }
    		cs

  • Object.prototype 에서 hasOwnProperty()를 호출하는 것도 또하나의 패턴이다.

    for(var i in man){
        if(Object.prototype.hasOwnProperty.call(man, i)){
            console.log(i,":", man[i]);
        }
    }

    		cs

  • 프로퍼티 탐색이 Object까지 멀리 거슬러 올라가지 않게 하려면 지연변수를 사용하여 이 메서드를 '캐쉬'하면 된다.

2.5 내장 생성자 프로토타입 확장하기 / 확장하지 않기

Object(), Array(), Function()과 같은 내장 생성자의 프로토타입을 확장하는 것은 좋으나, 코드가 예측에서 벗어나는 일이 많아지기 때문에 코드의 지속성이 심각하게 저해될수 있다.
루프안에서 hasOwnProperty()를 사용하지 않았다면 임의로 프로토타입에 추가된 프로퍼티가 튀어나와 혼동을 일으킬수도 있다.

생성자 프로토타입을 확장하려면 아래와 같은 조건을 모두 만족시켜야 한다.

  1. 해당 기능이 ECMAScript의 향후 버전이나 자바스크립트 구현에서 일관되게 내장 메서드로 구현될 예정이다. 예를들어 ECMAScript5 에 기술되었으나 아직 브라우저에 내장되지 않은 메서드라면 추가할 수 있다. 이 경우에는 유용한 메서드를 미리 정의하는 것이라고 할 수 있다.
  2. 이 프로퍼티 또는 메서드가 이미 존재하는지, 즉 이미 코드 어딘가에 구현되어 있거나, 지원브라우저 중 일부 자바스크립트 엔진에 내장되어 있는지 확인한다.
  3. 이 변경사항을 명확히 문서화하고 팀 내에서 공유한다.

2.6 switch 패턴

switch
  • if문은 프로그램이 실행되는 흐름에 분기(branch), else if를 사용해 다중분기를 수행
  • 모든 분기가 동일한 표현식의 값에 의해 결정될 때는 if(/else if)가 최선이 아니다.
  • switch문은 if문에서처럼 동일한 표현식이 여러 번 반복되는 문제점을 확실히 해결
- 자바스크립트 완벽가이드
다음 패턴을 따르면 switch문의 가독성과 견고성을 향상시킬 수 있다.
  • 각 case문을 switch문에 맞추어 정렬한다 (일반적인 중괄호 내 들여쓰기 규칙에서 벗어나는 방식이다.)
  • 각 case문 안에서 코드를 들여쓰기 한다.
  • 각 case문은 명확하게 break;로 종료한다.
  • break문을 생략하여 통과(fall-through)시키지 않는다. 그런 방법이 최선책이라는 확신이 있다면 해당 case에 반드시 기록을 남긴다. 코드를 읽는사람에게는 오류로 보일 수 있기 때문
  • 상용하는 case문이 하나도 없을 때도 정상적인 결과가 나올 수 있도록 switch문 마지막에는 default;문을 쓴다.

완전항등연산자 쓰기: 표현식과 타입을 모두 확인하는 === 와 !== 연산자를 사용

2.8 parseInt()를 통한 숫자 변환

  • parseInt()에서 보통 두번째 매개변수를 생략하는 경우가 많지만, 파싱할 문자열이 0으로 시작할 경우 문제가 생긴다.
  • 일관성 없고 예측을 벗어나는 결과를 피하려면 항상 기수 매개변수를 지정해주어야 한다.
    var month = "06",
        year = "09";
        month = parseInt(month, 10);
        year = parseInt(year , 10);
    		cs
parseInt()와 Number()의 차이

Number()

  • 문자열로 되어있는 숫자를 실제 숫자로 변환시켜주는 함수
  • 단, 문자열이 숫자로만 되어있어야함
  • 문자열안에 숫자와 문자가 섞여 있으면 NaN 값을 반환

parseInt()

  • 문자열을 정수로 변환
  • 자열안에 숫자와 문자가 섞여도 되지만 앞자리는 꼭 숫자여야함

2.15 출시 단계의 압축

  • 압축이란, 자바스크립트 코드에서 공백, 주석 및 기타 중요하지 않은 부분들으르 삭제 함으로써 서버에서 브라우저로 전송되는 파일크기를 감소시키는 공정
  • 페이지 로딩속도를 개선한다. 대개 용량이 반이상 줄어들 정도로 절약효과가 있어, 출시 준비가 완료된 스크립트에는 반드시 압축을 적용
  • 코드 압축이 페이지 성능을 향상시키는 데 중요하지만, 이것은 압축도구가 할일
